# Список тестов

## Класс Parser

## Этап 1: Лексические основы и первичные выражения (Primary Expressions)

- [ ] Распознать и распарсить целочисленный литерал `42`.
- [ ] Распознать и распарсить ноль `0`.
- [ ] Распознать и распарсить дробный литерал `3.14`.
- [ ] Распознать и распарсить идентификатор `myVar`.
- [ ] Распознать и распарсить идентификатор с цифрами и подчеркиванием `a_b1`.
- [ ] Распознать и распарсить выражение в скобках `(1 + 2)` как единое целое (узел `GroupedExpression`).

## Этап 2: Унарные операции (Unary Expressions)


- [ ] Распарсить унарный плюс `+5`.
- [ ] Распарсить унарный минус `-10`.
- [ ] Распарсить логическое отрицание `!flag`.
- [ ] Распарсить вложенные унарные операции `--5` (должно быть `-( -5)`).
- [ ] Распарсить унарную операцию над выражением в скобках `-(x + y)`.

## Этап 3: Бинарные операции — Приоритеты и Ассоциативность

### Возведение в степень (Power) — Правоассоциативное
- [ ] Распарсить простое возведение в степень `2 ^ 3`.
- [ ] **Критически важно:** Проверить правую ассоциативность: `2 ^ 3 ^ 4` должно парситься как `2 ^ (3 ^ 4)`, а не `(2 ^ 3) ^ 4`.

### Умножение, деление, остаток (Multiplicative) — Левоассоциативные
- [ ] Распарсить умножение `a * b`.
- [ ] Распарсить деление `x / y`.
- [ ] Распарсить остаток от деления `10 % 3`.
- [ ] Распарсить целочисленное деление `10 // 3`.
- [ ] Проверить левую ассоциативность: `8 / 2 / 2` должно парситься как `(8 / 2) / 2`.
- [ ] Проверить приоритет над сложением: `1 + 2 * 3` должно парситься как `1 + (2 * 3)`.

### Сложение и вычитание (Additive) — Левоассоциативные
- [ ] Распарсить сложение `1 + 2`.
- [ ] Распарсить вычитание `a - b`.
- [ ] Проверить левую ассоциативность: `1 - 2 - 3` должно парситься как `(1 - 2) - 3`.
- [ ] Проверить приоритет над сравнениями: `1 + 2 < 5` должно парситься как `(1 + 2) < 5`.

### Сравнения (Relational) — Левоассоциативные
- [ ] Распарсить оператор "меньше" `a < b`.
- [ ] Распарсить оператор "меньше или равно" `a <= b`.
- [ ] Распарсить оператор "больше" `a > b`.
- [ ] Распарсить оператор "больше или равно" `a >= b`.
- [ ] Проверить приоритет над операциями равенства: `a < b == false` должно парситься как `(a < b) == false`.

### Равенство (Equality) — Левоассоциативные
- [ ] Распарсить оператор "равно" `a == b`.
- [ ] Распарсить оператор "не равно" `a != b`.
- [ ] Проверить приоритет над логическим И: `a == b && c` должно парситься как `(a == b) && c`.

### Логическое И (AND) — Левоассоциативное
- [ ] Распарсить логическое И `a && b`.
- [ ] Проверить левую ассоциативность: `true && false && true` должно парситься как `(true && false) && true`.
- [ ] Проверить приоритет над логическим ИЛИ: `a && b || c` должно парситься как `(a && b) || c`.

### Логическое ИЛИ (OR) — Левоассоциативное
- [ ] Распарсить логическое ИЛИ `a || b`.
- [ ] Проверить левую ассоциативность: `true || false || true` должно парситься как `(true || false) || true`.

## Этап 4: Комплексные выражения и граничные случаи

- [ ] Распарсить сложное выражение с учетом всех приоритетов: `a + b * c ^ d - e / f`.
- [ ] Проверить, что скобки меняют приоритет: `(a + b) * c` должно парситься как `(a + b) * c`.
- [ ] Проверить взаимодействие унарных операций и приоритетов: `-a ^ b` (согласно грамматике, `unary_expr` имеет приоритет над `power_expr`, поэтому должно парситься как `(-a) ^ b`).
